# AstrBot 智能分段插件 (个人维护版)

在主 LLM 回复生成后先调用一个分段模型，把完整回复拆成更像真人聊天的多条消息；发送前只保留首段，首段发送完成后再按延迟异步补发剩余段。

> ⚠️ **声明与背景（个人维护版）**
> 
> 本项目为原插件 [exynos967/astrbot_plugin_smart_segmentation](https://github.com/exynos967/astrbot_plugin_smart_segmentation) 的个人维护分支。
> 
> **深度优化适配本地轻量模型（如 Qwen3.5-2B 等）**：
> 在本地运行 2B 级别的小参数模型时，通常会遇到指令遵循较弱（输出严格 JSON 格式时易崩溃）、容易复读 Few-Shot 示例、以及推理 Token 消耗敏感等痛点。为此，本项目对交互协议进行了深度重构（将高消耗的 JSON 格式优化为轻量级的管道符 `|` 分割形式），升级了提示词防线，并增加了一系列本地降级与防幻觉机制。这使得该插件不仅能运行在云端大模型上，在本地轻量级模型下也能提供更稳定的运行表现。

## 特性

### 核心能力 (继承自原版)
- 使用 AstrBot 原生 hooks：`on_llm_response`、`on_decorating_result`、`after_message_sent`。
- 只处理 LLM/Agent 的纯文本结果，不改变图片、语音、文件等非文本消息链。
- 支持括号动作/神态描述独立成段，并避免整段动作文本被继续拆分。

### 个人维护版增强特性
- **协议级数据传输优化**：将 LLM 交互协议重构为轻量级管道符（`|`）分隔文本，大幅减少了小参数模型因生成 JSON 格式崩溃的问题，同时大幅降低 Token 消耗。
- **大模型 Few-Shot 幻觉阻断**：新增 Prompt 示例文本的关键词拦截检测。当检测到小模型复读示例（如“今天发工资了”）时，自动熔断并降级至本地切分，有效防止幻觉文本外泄。
- **非自然语言拦截与本地切分兜底**：在发送请求前自动拦截 JSON、代码块等非自然语言文本 [1]；在网络超时、交互异常或缓存失配时，无缝降级为基于标点与括号规则的本地纯逻辑切分，确保消息正常发出。
- **Markdown 重度文本保护**：自动识别排版复杂的 Markdown 笔记（包含标题、列表等特征），并放弃句末符号切分，改为按双换行（段落）切分，保护原文排版不被破坏。
- **代码函数避让机制**：增加语法前置哨兵，当括号左侧紧邻字母/数字等字符时自动判定为代码语法，防止切碎代码函数（如 `split()`）。
- **颜文字与连续标点粘连保护**：防止 `？！` 等连续标点被切碎；针对字母颜文字（如 `qwq`）和被括号包裹的情绪描述符号（如 `(>_<)`）提供粘连保护，避免产生孤立的符号段落。
- **控制符过滤与空气泡消除**：引入无意义 Markdown 分割符（如 `---`、`***`）清洗机制，防止因其独立成段导致发送空消息气泡，降低群聊风控拦截概率。
- **括号平衡强制自愈**：优化了括号平衡逻辑，由跨段合并机制改为单段内括号数量缺省强制自动补齐，消除了后续段落因丢失括号被无限吸入合并的缺陷。

## ⚙️ 测试环境与推荐配置（重要）

### 1. 风格兼容性声明
> ⚠️ **注意**：在开发与日常测试过程中，**维护者始终采用 `natural`（自然）分段风格**。由于未能对其他分段风格（如 `conservative` 或 `active`）进行充分的回归测试，我不确定切换风格后是否会产生意料之外的分段异常。**强烈建议保持使用 `natural` 风格以确保稳定性**。

### 2. 针对本地小模型（如 Qwen3.5-2B）的推荐配置参考
如果你的分段模型运行在本地（例如通过 Ollama 运行 `qwen3.5:2b`），以下是经过维护者实测并长期使用的稳定配置：

*   **分段模型 Provider ID**：`ollama/qwen3.5:2b`
*   **分段风格**：`natural`（强烈推荐）
*   **分段模型温度（Temperature）**：`0`（建议设为0。对于本地小模型，降低随机性是防止格式崩溃最有效的手段 [1]）
*   **最大分段数量**：`12`
*   **分段模型超时时间 (秒)**：`20`（本地运行由于硬件性能或首字延迟加载较慢，建议放宽至 20 秒）
*   **补发基础延迟 (秒)**：`2`
*   **按字符增加延迟 (秒/字符)**：`0.05`
*   **单段最大补发延迟（秒）**：`3`

## 配置说明（全局列表）

| 配置项 | 说明 | 默认值 |
| --- | --- | --- |
| `enabled` | 启用/禁用插件 | `true` |
| `provider_id` | 分段模型 Provider ID，留空使用当前会话模型 | `""` |
| `style` | `natural` / `conservative` / `active` | `natural` |
| `min_length` | 文本短于该长度不分段 | `15` |
| `max_segments` | 单次最多分段数量 | `8` |
| `temperature` | 分段模型温度 | `0.3` |
| `max_tokens` | 分段模型最大输出 token | `600` |
| `timeout_seconds` | 分段模型超时时间 | `12.0` |
| `delay_base` | 补发基础延迟（秒） | `0.35` |
| `delay_per_char` | 按字符增加的延迟（秒/字符） | `0.015` |
| `delay_max` | 单段最大补发延迟（秒） | `1.2` |

> 💡 **使用建议**：建议关闭 AstrBot 内置的 `platform_settings.segmented_reply.enable`，避免其与本插件产生重复分段。
