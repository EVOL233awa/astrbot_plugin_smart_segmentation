The format is based on [Keep a Changelog](https://keepachangelog.com/en/2.0.0/), and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.0.0] - 2026-07-12

### Added
- **非自然语言拦截与本地切分兜底**：在发送请求至 LLM 前，自动拦截 JSON 和代码块等非自然语言文本；在网络超时或缓存失配时，无缝降级为基于标点与括号规则的本地纯逻辑高精切分，确保消息正常发送。
- **Markdown 重度文本保护**：自动识别排版复杂的 Markdown 笔记（包含标题、列表等特征），并放弃句末符号切分，改为按双换行（段落）切分，防止破坏原文排版。
- **控制符过滤与空气泡消除**：引入无意义 Markdown 分割符（如 `---`、`***`）清洗机制，防止因其独立成段导致发送空消息气泡，并降低群聊风控拦截概率。
- **大模型 Few-Shot 幻觉阻断**：新增 Prompt 示例文本的关键词拦截检测。当检测到模型复读示例（如“今天发工资了”）时自动熔断并降级至本地切分，有效防止幻觉文本外泄。
- **颜文字与连续标点粘连保护**：在切分逻辑中实现标点符号聚类，防止 `？！` 等连续标点被切碎。针对字母颜文字（如 `qwq`）和被括号包裹的情绪描述符号（如 `(>_<)`）提供粘连保护，避免产生孤立的符号段落。

### Changed
- **自愈式消息还原与三级降级检索**：缓存管道升级为“精确哈希匹配 → 顺序贪婪组合自愈（适配多轮 Tool Loop 拼接场景）→ 本地高精兜底”，大幅提升在复杂交互链路下的消息分段还原精确率。
- **协议级数据传输优化**：将 LLM 交互协议由高耗 Token 的 JSON 数据格式重构为轻量级管道符（`|`）分隔文本，大幅减少小参数模型格式崩溃问题并降低 Token 消耗 [1]。
- **括号平衡强制自愈**：重写括号平衡逻辑，由跨段合并机制改为单段内括号数量缺省强制自动补齐，消除了后续段落因丢括号被无限吸入合并的粘连缺陷。
- **超限截断物理粘连优化**：在超出最大分段数量触发尾部强行截断合并时，词语拼接符号由无缝拼接调整为空格拼接，防止前后词语发生物理粘连。
- **提示词工程防线升级**：在生成 Prompt 中引入显式负面约束（Negative Constraints）并扩充Few-Shot示例，降低了小参数模型的指令遵循门槛。

### Fixed
- **推理过程泄露修复**：支持非闭合容错正则，确保在 LLM 推理过程发生 Token 截断的情况下，仍能剥离未闭合的 `<thinking>` 思考标签。
- **单引号与非标 JSON 解析容错**：引入多级保底解析与尾部逗号清洗逻辑，兼容非标准 JSON 以及包含英文单引号的特殊输入。
- **代码函数避让机制**：增加语法前置哨兵，当括号左侧紧邻字母/数字等字符时自动判定为代码语法，防止切碎代码函数（如 `split()`）。
- **会话状态机残留清理**：在贪婪拼接匹配到缓存后即刻执行销毁消费，避免在多轮 Tool Call 并发场景下造成分段残留。
